Accessibility

Accessibility Statement

Our commitment to making KPBConstruction.com accessible to everyone.

Last reviewed: April 1, 2026

Partially Conformant

Our Commitment

KPB Construction LLC is committed to ensuring that our website is accessible to people with disabilities. We strive to meet the Web Content Accessibility Guidelines (WCAG) 2.1 Level AA, as referenced in Section 508 of the Rehabilitation Act and the Americans with Disabilities Act (ADA).

Accessibility is an ongoing effort. We regularly review our website and take steps to improve the experience for all visitors, regardless of ability or assistive technology used.

Conformance Status

The website is partially conformant with WCAG 2.1 Level AA. "Partially conformant" means that some parts of the content do not fully conform to the accessibility standard. We are actively working to address known gaps.

Accessibility Features

The following features are currently implemented on this website:

Navigation & Structure

  • Skip to content link: A "Skip to main content" link appears at the top of each page, allowing keyboard users to bypass the navigation bar.
  • Semantic HTML landmarks: Pages use <nav>, <main>, <footer>, and ARIA landmark roles to support screen reader navigation.
  • Logical heading hierarchy: Content is structured with h1 through h4 headings in a logical order.
  • Language declaration: The page language is declared as English (lang="en").

Keyboard Accessibility

  • Focus indicators: All interactive elements display a visible focus outline when navigated by keyboard.
  • FAQ accordion: FAQ items use <button> elements with aria-expanded attributes, fully operable by keyboard.
  • Portfolio filters: Category filter buttons are standard HTML <button> elements, fully keyboard-accessible.
  • Forms: All form inputs are associated with visible labels via for/id attributes, and forms are fully operable by keyboard.

Images & Media

  • Alternative text: Portfolio and content images include descriptive alt attributes.
  • Decorative elements: Decorative emoji icons are marked with aria-hidden="true" to avoid unnecessary screen reader verbosity.

Color & Contrast

  • Body text on light backgrounds meets the WCAG 4.5:1 contrast ratio requirement for normal text.
  • Primary call-to-action buttons use sufficient color contrast between text and background.

Cookie Consent

  • The cookie consent banner is keyboard accessible and includes appropriate ARIA attributes (role="dialog", aria-label).

Known Limitations

We are aware of the following accessibility gaps and are working to address them:

  • Portfolio lightbox (Frankton page): The image lightbox modal may not fully trap keyboard focus while open. We are evaluating a focus-trap solution.
  • Portfolio cards (Carmel page): Clickable portfolio cards use onclick handlers; we are adding role="button" and keyboard event support.
  • Mobile hamburger menu: The mobile navigation toggle does not currently announce its open/closed state via aria-expanded. This will be addressed in a future update.
  • Live chat widget (Tawk.to): The Tawk.to chat widget is a third-party component. Accessibility of the widget itself is subject to Tawk.to's implementation. We encourage users who experience issues to contact us directly by phone or email instead.

Assistive Technologies Tested

We test our website with the following assistive technologies:

  • NVDA screen reader on Windows (Chrome)
  • VoiceOver on macOS (Safari)
  • Keyboard-only navigation (Tab, Shift+Tab, Enter, Space, Escape)
